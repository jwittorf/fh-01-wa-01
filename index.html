Grundlegendes zu HTML und CSS

HTML ist eine Auszeichnungssprache und dient der Strukturierung von Text und allen weiteren Inhalten, die auf einer Webseite enthalten sein können. Dazu gehören auch Bilder, Listen, Tabellen, Verlinkungen und vieles mehr. Die Verwendung von sogennanten <tags> für die zuvor genannten Elemente (Auflistung ist nicht vollständig) hilft einem Browser, der für die Darstellung/das Rendering der Seite zuständig ist, und anderen Maschinen wie Suchmaschinen oder Screenreadern die Inhalte der Webseite richtig zu verstehen und darauf basierend entsprechende Operationen durchzuführen. Diese kann die Darstellung, die Bewertung/Gewichtung oder die Betonung sein, dies obliegt der Maschine und ihrem Autor.

HTML obliegt gewissen Regeln, die weltweit standardisiert sind und in der DTD hinterlegt werden. Der Autor gibt damit an, welchen Regeln er in seinem Dokument folgt. Mittels eines Validierungsdienstes kann überprüft werden, ob die Regeln eingehalten werden. Bei einer Nichteinhaltung kann es zu Problemen bei der Verarbeitung des Dokuments kommen, sodass z. B. ein Browser die Seite nicht korrekt darstellt oder andere externe Programme die Datei nicht akzeptieren.

Idealerweise erfolgt zur besseren Wartbarkeit und Performance einer Webseite bzw. Website (Verbund mehrerer Webseiten, quasi ein Portal oder eine ganze Präsenz) eine strikte Trennung von Inhalt und Gestaltung. Im HTML steht nur der Inhalt, die Gestaltung wird in CSS ausgelagert, idealerweise in eine eigene Datei, sodass mehrere Seiten die gleiche Gestaltung verwenden können.

Mit CSS wird die Gestaltung der HTML-Elemente bzw. des Dokuments, in dem die Datei eingebunden festgelegt, man könnte CSS daher auch als Konfigurationssprache bezeichnen. Das Zusammenspiel ist ähnlich eines Word-Dokuments mit Formatvorlagen, wobei HTML den Inhalt und CSS die Formatvorlagen und darin enthaltenen Anweisungen darstellt - ändert man die Formatvorlage, ändert sich auch der Inhalt, auf dem die Formatvorlage angewendet ist. CSS verfügt über sehr mächtige Werkzeuge wie Operatoren oder Grafik- und Animationsanweisungen.

In der modernen Webentwicklung werden zunehmend CSS-Preprozessoren wie SASS/SCSS oder LESS eingesetzt, die das Schreiben von (besonders komplexen) CSS-Dateien erleichtert, beschleunigt und weitere Funktionen wie die Nutzung von Funktionen, Skripten und Variabeln zulässt. Die Nutzung von Variablen gewährleistet so z. B. auch eine konsistente Nutzung der gleichen Werte wie Farben, Schriften oder Abständen. Die Preprozessoren "überwachen" eine Datei oder einen ganzen Ordner auf Änderungen und kompilieren in (Milli)-Sekunden eine vollständige und CSS-Datei, die "ganz normal" weiterverarbeitet werden kann. Sie stellen also eine Unterstützung für die Entwickler dar, ohne negative Auswirkung auf die Besucher der Seite. Die Auswirkungen sind sogar positiv, da z. B. bei der Nutzung von CSS-Frameworks wie "Twitter Bootstrap" die einzelnen Komponenten direkt an die Bedürfnisse des Autors angepasst und so eine eigene, individuelle Version des Frameworks mit eigenen Werten (durch Nutzung von Variabeln) generiert wird. Diese müssen nicht durch zusätzlichen, eigenen CSS-Code überschrieben werden, was wiederum zu einer Verbesserung der Performance führt. Außerdem kann der Entwickler Funktionen des Frameworks nutzen und erweitern.

In speziellen Fällen wird ein Teil des CSS-Codes in das <head>-Element geschreiben, um die Wartezeit für das Herunterladen der Datei zu vermeiden. Dieser Sonderfall dient der Performance-Optimierung und enthält nur Stil-Informationen, die für die Darstellung der Elemente benötigt werden, die beim Laden der Seite im sichtbaren Bereich erscheinen (auch bekannt als Viewport). Zur Generierung einer solchen Datei kann z. B. das Node.js-Plugin "Penthouse" verwendet werden, welches unter Nutzung des "Phantom"-Browsers und entsprechender Konfiguration der Viewports, die benötigten CSS-Anweisungen aus einer Datei extrahiert und z. B. in eine eigene Datei schreibt, deren Inhalt ausgelesen und in den <head>-Bereich geschrieben werden kann.

Überschriften in HTML

Überschriften werden in HTML über die Elemente <h1> bis <h6> dargestellt. Sie definieren die "Wichtigkeit" der Überschrift in absteigender Reihenfolge und dienen der Strukturierung (ähnlich wie in einem Buch mit Kapitelüberschriften und Unterkaptieln). Weiterhin können sie von Suchmaschinen und Screenreadern genutzt werden, um den Inhalt der Seite besser zu verstehen oder etwas anders zu betonen, um einen Kontext herzustellen.

Hinweis: eine ganze Zeit lang galt es gängige Praxis für Suchmaschinen (besonders Google) nur eine <h1>-Überschrift zu verwenden, um zu verdeutlichen, was das Hauptthema (Keyword) der Seite ist. Weitere Aspekte waren auch die Nutzung von Document Outlines. Mit den neuen Tags von HTML5 ist diese Diskussion jedoch obsolet und es wird sogar empfohlen, in manchen Fällen mehrere <h1>-Überschriften zu verwenden. Wie so oft kommt es auch wieder auf den Kontext und die korrekte und semantisch richtige Anwendung an, weitere Informationen sind diesem Blog-Artikel zu entnehmen.

Attributsselektoren

Attributselektoren erlauben es, Elemente nur bei einer bestimmten Attributkombination dann anzusprechen.

Name Syntax Beschreibung HTML-Beispiel
Attributpräsenz [attribut] Ein Element wird nur angesprochen, wenn das angegebene Attribut vorhanden ist. <button disabled>Deaktivierte Schaltfläche</button>
Attributwert [attribut="wert"] Ein Element wird nur angesprochen, wenn es ein Attribut mit einen bestimmten Wert besitzt. <a rel="external">Externer Link</a>
Teilübereinstimmung [attribut^="wert"] [attribut$="wert"] [attribut*="wert"] Es können Elemente angesprochen werde die, mit einer bestimmten Zeichenkette beginnen, mit einer bestimmten Zeichenkette enden oder eine bestimmte Zeichenkette enthalten. <a href="http://aol.com"> <a href="http://db.de"> <a href="http://wiki.org">

Listen

In HTML wird hauptsächlich zwischen geordneten und ungeordneten Listen unterschieden. Ungeordnete Listen werden z. B. für Menüs und Navigationsbäume verwendet. In Definitionslisten werden Definitionen beschrieben.

Ungeordnete Listen

  • haben keine semantische
  • Reihenfolge.
  • Sie können verschachtelt werden,
    • sodass weitere
    • Ebenen dargestellt
    • werden können.
  • Ungeordnete Listen werden mit einem <ul>-Tag umschlossen,
  • die einzelnen Listenpunkte
  • werden mit je einem <li>-Tag umschlossen.

Geordnete Listen

  1. verhalten sich ähnlich,
  2. sie können per CSS
    1. sogar automatische
    2. Aufzählungszeichen wie Zahlen oder Buchstaben
    3. erhalten, der Unterschied ist nur in der Semantik.
  3. Diese Listen werden mit einem <ol>-Tag umschlossen.
